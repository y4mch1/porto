Local RAG: Intelligent Document Analysis

🚧 Work in Progress: This project is currently under active development. The documentation below reflects the current backend architecture and logic implementation (v0.1).
"A privacy-focused document summarization tool powered by Llama 3.1. It uses K-Means Clustering to intelligently select the most relevant document chunks, ensuring comprehensive context coverage without hitting token limits."

Project Overview

This tool is designed to analyze complex PDF documents locally without sending data to the cloud. Unlike standard RAG (Retrieval-Augmented Generation) systems that simply fetch the "top-k" similar chunks, this system employs Semantic Clustering.

By clustering document embeddings, the system ensures that the information fed into the LLM is diverse and representative of the entire document, reducing redundancy and improving summary quality.

System Architecture & Pipeline

The system follows a strict ETL (Extract, Transform, Load) pipeline optimized for local execution:

1. Ingestion PyPDFLoader reads raw PDF files.
→
2. Chunking Recursive Character Splitter (2k tokens).
→
3. Embedding BAAI/bge-base-en (HuggingFace).
→
4. Clustering K-Means selection (The "Brain").
→
5. Inference Llama 3.1 via Ollama.

Why K-Means Clustering?

In the code logic EmbeddingsClusteringFilter, instead of just taking the first 5 pages, the system:

  • Converts all text chunks into vector embeddings.
  • Groups these vectors into clusters based on semantic meaning.
  • Selects the "Centroid" (most representative chunk) from each cluster.
  • Result: The LLM receives a context that covers different topics within the PDF, rather than repeating the same topic 5 times.

Key Technical Features

1. Local Privacy First

The system runs entirely offline using Ollama. No API keys are required, and sensitive documents (e.g., financial reports, medical records) never leave the local machine.

2. Strict Hallucination Control

The prompt engineering includes strict guardrails to prevent the LLM from making things up:


# System Prompt Logic
"guidelines": 
1. Jawab hanya berdasarkan teks konteks.
2. Pertahankan penamaan tabel dan satuan angka.
3. Jika data tidak ada, tulis 'Informasi tidak ditemukan'.
4. Tutup jawaban dengan bagian 'Validasi'.

3. Dynamic Resource Handling

The script automatically detects hardware capabilities:

  • CUDA Support: Automatically utilizes GPU if available for faster embedding.
  • Fallback: Gracefully degrades to CPU mode if no GPU is found.
