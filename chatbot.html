Local RAG: Intelligent Document Analysis

🚧 Work in Progress: This project is currently under active development. The documentation below reflects the current backend architecture (v0.1) integrating Markdown Extraction, FAISS retrieval, and K-Means Clustering for summarization.
"A privacy-focused document analysis tool optimized for multilingual and complex reports. It features dual-mode processing: precision Q&A via FAISS vector retrieval, and comprehensive summarization powered by K-Means semantic clustering."

System Architecture & Pipeline

The system follows a robust ETL (Extract, Transform, Load) pipeline optimized for local execution and maximum contextual understanding:

1. Ingestion pymupdf4llm converts PDF directly to Markdown format.
→
2. Chunking MarkdownTextSplitter (2500 chars, 300 overlap).
→
3. Embedding multilingual-e5-base via HuggingFace.
→
4. Indexing & Logic FAISS (Q&A Retrieval)
+ K-Means (Summarization).
→
5. Inference Llama 3.1 via Ollama (temperature=0).

Key Technical Features

1. Dual Analytical Modes

The system intelligently routes the user request based on the command provided in the CLI:

  • Q&A Mode (RAG via FAISS): By asking a specific question, the system retrieves the Top-8 (MAX_RAG_DOCS) most relevant chunks from the FAISS vector database to answer accurately.
  • Summary Mode (K-Means Clustering): Triggered by the "ringkas" keyword. Instead of grabbing the first 5 pages, it clusters all text embeddings into distinct semantic groups (up to 5 clusters). It selects the "Centroid" of each group, ensuring the LLM summarizes a diverse representation of the entire document.

2. Local Privacy & Hardware Optimization

Runs entirely offline utilizing Ollama. The script dynamically detects hardware capabilities:

  • CUDA Auto-Detection: Transparently switches to GPU for blazing-fast intfloat/multilingual-e5-base embedding generation if a capable NVIDIA card is detected.
  • Fallback Mode: Gracefully defaults to CPU execution without breaking the pipeline.

3. Strict Hallucination Control (Prompt Engineering)

To analyze complex financial and technical reports, the prompt is equipped with stringent formatting and factual guardrails:


# Guidelines Logic
1. Jawab HANYA berdasarkan teks konteks; bila data tidak ada, tulis 'Informasi tidak ditemukan pada konteks'.
2. Pertahankan penamaan tabel, label grafik, dan satuan angka persis seperti di sumber.
3. Saat menyebut angka, sebutkan juga referensi asalnya agar mudah dilacak.
4. Jika ada tabel, salin ulang dalam format tabel Markdown tanpa mengubah/menambah data.
5. Tutup jawaban dengan bagian 'Validasi' yang menjelaskan apakah semua permintaan pengguna terpenuhi dari konteks.

4. Structural Output Formatting

Responses are heavily structured depending on the task. A Q&A output must contain Jawaban Inti, Referensi Konteks, and Validasi, whereas a Summary output requires Ringkasan Utama and Detail Numerik & Tabel.
